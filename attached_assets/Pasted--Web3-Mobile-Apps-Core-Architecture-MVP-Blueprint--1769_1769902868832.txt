ر Web3 وMobile Apps.
مطلوب منك تصميم الهيكلة الأساسية (Core Architecture + MVP Blueprint)
لتطبيق تعدين عملة مشفرة يهدف إلى الترويج للسياحة العالمية.

⛏️ فكرة التطبيق:
تطبيق تعدين يعتمد على تفاعل المستخدم الحقيقي (إعلانات + فيديوهات سياحية + مهام)
وليس التعدين الحسابي التقليدي.

━━━━━━━━━━━━━━━━━━
1️⃣ نظام التعدين (Mining System)
━━━━━━━━━━━━━━━━━━
- التعدين يتم كل 4 ساعات (Mining Cycle).
- يجب على المستخدم الضغط يدويًا لبدء التعدين.
- عند كل ضغط:
  - عرض إعلان إجباري مدته 60 ثانية (لا يمكن تخطيه).
  - لا يتم احتساب التعدين إلا بعد مشاهدة الإعلان كاملًا.
- لكل دورة تعدين:
  - معدل تعدين أساسي Base Mining Rate.

━━━━━━━━━━━━━━━━━━
2️⃣ تعزيز التعدين (Boost System)
━━━━━━━━━━━━━━━━━━
- يمكن للمستخدم زيادة نسبة التعدين في نفس دورة التعدين عبر:
  - مشاهدة فيديو يوتيوب سياحي لدولة معينة.
  - شرط مشاهدة الفيديو كاملًا (Full Watch Validation).
  - بعد انتهاء الفيديو يظهر سؤال.
  - الإجابة الصحيحة موجودة داخل الفيديو فقط.
  - عند الإجابة الصحيحة:
    - منح Boost (زيادة عدد العملات في هذه الدورة فقط).

━━━━━━━━━━━━━━━━━━
3️⃣ المهام اليومية (Daily Tasks System)
━━━━━━━━━━━━━━━━━━
- كل مستخدم لديه 20 مهمة يوميًا.
- كل مهمة:
  - نشاط بسيط (مشاهدة إعلان، قراءة معلومة سياحية، تفاعل…).
  - تمنح مكافأة عملات تضاف إلى الرصيد.
- المهام تُعاد يوميًا (Reset كل 24 ساعة).

━━━━━━━━━━━━━━━━━━
4️⃣ نظام تقسيم المعدنين (Anti-AdBan System)
━━━━━━━━━━━━━━━━━━
- لتفادي حظر حسابات الإعلانات:
  - يتم تقسيم المستخدمين إلى مجموعات.
- الحد الأقصى للمجموعات: 10.
- مثال:
  - 1,000,000 مستخدم → 10 مجموعات × 100,000 مستخدم.
- كل مجموعة:
  - لها جدول مهام مختلف.
  - مهام موزعة على فترات 4 ساعات.
  - لا تتفاعل جميع المجموعات مع الإعلانات في نفس الوقت.
- الهدف:
  - محاكاة السلوك البشري الطبيعي.
  - منع اكتشاف النشاط كـ Bot Traffic.

━━━━━━━━━━━━━━━━━━
5️⃣ الملف الشخصي (User Profile)
━━━━━━━━━━━━━━━━━━
- لكل مستخدم:
  - معرف فريد User ID.
  - رصيد العملات.
  - تاريخ التعدين.
  - حالة المهام اليومية.
  - المجموعة التي ينتمي إليها.
- دعم KYC لاحقًا:
  - حقل حالة KYC (Not Started / Pending / Verified).
  - بدون تنفيذ فعلي للـ KYC في المرحلة الأولى.

━━━━━━━━━━━━━━━━━━
6️⃣ منع الغش (Anti-Fraud & Validation)
━━━━━━━━━━━━━━━━━━
- منع التعدين المتكرر خارج التوقيت.
- منع تجاوز الإعلان أو الفيديو.
- التحقق من:
  - الوقت.
  - الجلسة.
  - السلوك المتكرر المشبوه.
- تسجيل Logs أساسية لكل عملية تعدين.

━━━━━━━━━━━━━━━━━━
7️⃣ المطلوب منك كمخرجات
━━━━━━━━━━━━━━━━━━
أريد منك:
1. تصور معماري عام (Frontend / Backend / Services).
2. نموذج قاعدة بيانات أولي (Entities + Relations).
3. Flow واضح:
   - Mining Flow
   - Ads Flow
   - Video Boost Flow
   - Daily Tasks Flow
4. اقتراح تقنيات مناسبة (Web / Mobile / Backend).
5. تحديد ما هو MVP وما يؤجل للمرحلة الثانية.
6. مراعاة قابلية التوسع إلى ملايين المستخدمين.
7. شرح مختصر لكل جزء بدون الدخول في كود.

❗ اعتبر أن هذا مشروع حقيقي قابل للإطلاق التجاري.
